API Overview

Enterprise-grade audio processing and speech recognition API

Base URL

http://35.153.178.201:8080
All API requests should be made to this base URL. Authentication may be required for certain endpoints.

API Architecture

The AI Audio API provides a comprehensive solution for audio processing, transcription, and analysis. Built with a microservices architecture, it supports:

  • Audio transcription
  • Multi-language support
  • Sentiment and emotion analysis
  • User and company management
  • Role-based access control
Note: This is a beta version of the API. Some features may change in future releases.

Authentication

Secure access to API endpoints

🔐 Login & Token Generation

To access protected endpoints, you first need to obtain an access token.

POST /auth/token

Request Body

{ "username": "superadmin", "password": "superadminpassword" }

Response

{ "access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...", "token_type": "bearer" }
Usage: Include this token in the Authorization header for all subsequent requests: Authorization: Bearer YOUR_ACCESS_TOKEN

👤 Get User Profile

Retrieve current user information using your access token.

GET /auth/me

Headers

Authorization: Bearer YOUR_ACCESS_TOKEN

Response

{ "username": "john_doe", "email": "user@example.com", "full_name": "John Doe", "position": "Sales Agent", "phone": "+1234567890", "id": "user_123", "is_active": true, "role": { "id": "role_456", "name": "agent" }, "company_id": "company_789", "company_name": "ABC Corp", "supervisor_id": "user_456" }

📋 Quick Start Guide

Step 1: Get Access Token

curl -X POST "http://35.153.178.201:8080/auth/token" \ -H "Content-Type: application/json" \ -d '{"username": "superadmin", "password": "superadminpassword"}'

Step 2: Use Token in API Calls

curl -X GET "http://35.153.178.201:8080/auth/me" \ -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
Security Note: Never expose your access token in client-side code. Always store tokens securely and use HTTPS in production.

📋 RBAC Quick Reference

System Roles (Cannot be modified/deleted)

SuperAdmin
Full system access across all companies
Admin
Full access within assigned company
Supervisor
Can manage agents under supervision
Agent
Standard user with basic permissions

Common Permission Types

User Management
manage_users, view_users, deactivate_users
Role Management
manage_roles, view_roles, assign_roles
Company Management
manage_companies, view_companies
Audio Processing
upload_audio, transcribe_audio, analyze_audio

view_content, moderate_content, approve_content
Workflow Tip:
  1. View available permissions with GET /permissions
  2. Create role with POST /roles
  3. Assign permissions with PUT /roles/{role_id}
  4. Assign role to users during user creation or update
Security Best Practices:
  • Follow principle of least privilege - assign only necessary permissions
  • Regularly audit role assignments and permissions
  • Use custom roles for specific business functions
  • Test permission changes in a non-production environment first

  • Companies can be moved between groups without data loss
Deletion Impact:
  • Companies will lose group configuration inheritance
  • API limits will revert to individual company settings
  • Historical data and analyses are preserved
  • Companies must be reassigned to new groups if needed

Process audio file for transcription and analysis using speaker diarization.
